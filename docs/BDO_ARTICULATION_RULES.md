# BDO 奏法匹配规则草案

状态：2026-07-10。

这份规则用于 GUI 的“优化 MIDI”、奏法建议和 FX 提示。优化窗口会先生成预览报告，只有用户点“应用优化”才会写入音符级 `ntype`。黑色沙漠公开资料只说明弗罗凯特拉等乐器有更丰富的音色与调制能力，具体 `ntype` 的声音细节主要来自我们在游戏内保存文件、截图和反解结果；因此未完全确认的项目会标为“需人工验证”。

## 通用判断

| 奏法 | 程序推荐条件 | 备注 |
| --- | --- | --- |
| 延音 | 默认值；旋律线、长音、和声铺底、无法确定时 | 最安全，导出不应主动替换 |
| 剪切 | 音符很短、音符之间有明显空隙、断奏型重复 | 接近 staccato / detached |
| 标签 | 只作为强调或已验证标记使用 | BDO 实际声音仍需人工验证 |
| 向上滑动 | 当前音后接更高音，间隔 1-4 半音，且音符连接或间隔很短 | 适合上行倚音、滑入目标音 |
| 滑弦下降 | 当前音后接更低音，间隔 1-4 半音，且音符连接或间隔很短 | 适合吉他、贝斯下滑或收尾 |
| 滑音 / 滑音上升 | 连续快速级进、琶音、跨越超过 5 半音的装饰性滑奏 | 更适合竖琴、贝斯、合成音色 |
| 弱音 | 吉他/贝斯短音、低到中等力度、伴奏节奏型、重复低音 | 接近 palm mute / muted |
| 泛音 | 高音区、稀疏点缀、低到中等力度、非密集主旋律 | 不适合整轨密集套用 |
| 三连音 | 局部节奏明显为一拍内三等分或连续三连音装饰 | 只在支持的乐器上推荐 |
| 颤音小调 | 长音或反复邻音；邻音间隔 1 半音 | 接近 minor trill |
| 颤音大调 / 大调颤音 | 长音或反复邻音；邻音间隔 2 半音 | 接近 major trill |
| 颤音 / 颤音 2/3/4 | 长音、持续音色变化、同音快速重复 | 多个变体需游戏内继续确认音色差别 |
| SusPiano / SusMezzoForte / SusForte | 管乐长音，按力度分层：弱、中、强 | 建议按 velocity 分配，而不是整轨固定 |
| 延音踏板 | 钢琴和声延续、MIDI 有 CC64、或同和弦重叠保持 | 只用于钢琴系 |
| 维持滤波器 | 玛勒尼恩合成铺底、长音、持续纹理 | 需人工验证具体音色 |
| 滤波铜管 | 玛勒尼恩明亮或铜管感长音，高力度更适合 | 需人工验证具体音色 |
| 拍弦 | 贝斯高力度短音、funk 节奏、八度跳进 | 不应套到长音 |
| X-音符 | 贝斯死音/鬼音，极短、低力度、节奏填充 | 可作为静音打击感，不保证有明确音高 |
| FX(C2~G2) | 电吉他特效触发音，音高在 C2-G2 或用户明确选择 | 不用于普通旋律自动转换 |
| 基本 | 玛勒尼恩部分乐器的基础变体 | 声音差别需游戏内验证 |

## 乐器分组规则

### 管乐器

适用乐器：长笛、竖笛、圆号。

- 长线旋律默认用“延音”。
- 短促断句、跳音用“剪切”。
- 上行小滑音用“向上滑动”。
- 长音上带邻音装饰时用颤音类：半音邻音优先“小调”，全音邻音优先“大调”。
- 圆号和竖笛的 `SusPiano/SusMezzoForte/SusForte` 应根据力度分层，不建议整轨固定为一个 Sus。

### 弦乐器

适用乐器：吉他、竖琴、小提琴、低音提琴、玛勒尼贝斯。

- 小提琴长旋律默认“延音”，短促弓法可推荐“剪切”。
- 小提琴长音装饰可推荐颤音类，但不同 BDO 颤音变体需要继续截图/试听确认。
- 吉他和贝斯节奏型短音可推荐“弱音”。
- 吉他和贝斯上/下行半音或全音连接可推荐滑动类。
- 贝斯高力度短音可推荐“拍弦”，低力度极短装饰音可推荐“X-音符”。
- 竖琴大范围快速连续音可推荐“滑音”；明确大/小三和弦块可推荐“大调和弦/和弦小调”。

### 键盘乐器

适用乐器：钢琴。

- 钢琴普通音默认“延音”。
- MIDI CC64 踏板、或和声明显需要保持时，推荐“延音踏板”。
- 钢琴暂不自动使用其他奏法。

### 电吉他

适用乐器：银色水波、高速路、赫赛德兰。

- 普通旋律或和弦默认“延音”。
- 节奏切分、短促伴奏可推荐“弱音”。
- 高音区点缀可推荐“泛音”。
- 长音或快速同音重复可推荐“颤音”。
- `FX(C2~G2)` 只作为特效触发，不能自动把普通音符改成 FX。

### 玛勒尼恩合成乐器

适用乐器：波纹行星、幻象树、秘密笔记、三明治。

- 合成铺底和旋律默认“延音”或“基本”。
- 长音、持续纹理可推荐“维持滤波器”。
- 明亮铜管感或高力度长音可推荐“滤波铜管”。
- 颤音变体、标签、剪切需要继续游戏验证，第一版只提示，不自动应用。

### 打击乐器

- 架子鼓套装使用 pitch `48-64`，`63 SnrRollS` 和 `64 SnrRollL` 是鼓滚。
- 非鼓滚打击乐应保持短时值，避免游戏内重叠不出声。
- `GM 37 Side Stick` 推荐到 `49 SnrSide`。
- 手鼓、钹、手碟目前只有样本音高，不足以自动判断完整音域，应显示“需人工验证”。

## 可程序化的第一版启发式

这些规则可以进入 GUI 的“推荐奏法”，默认只给建议。映射表登记不等同于游戏内 A/B 验证；未建立验证矩阵前不得自动写入音符 `ntype`。

| 条件 | 推荐 |
| --- | --- |
| note duration <= 120 ms 且同轨短音比例高 | 剪切或弱音 |
| velocity >= 105 且乐器是玛勒尼贝斯，duration <= 180 ms | 拍弦 |
| velocity <= 55 且乐器是玛勒尼贝斯，duration <= 90 ms | X-音符 |
| 当前音到下一音上行 1-4 半音，gap <= 80 ms | 向上滑动 |
| 当前音到下一音下行 1-4 半音，gap <= 80 ms | 滑弦下降 |
| 持续音 duration >= 500 ms，局部存在 1 半音邻音来回 | 颤音小调 |
| 持续音 duration >= 500 ms，局部存在 2 半音邻音来回 | 颤音大调 |
| MIDI 有 CC64，或钢琴同和弦重叠保持 | 延音踏板 |
| pitch 在高音区且音符稀疏，乐器支持泛音 | 泛音 |
| 三个近似等距音符落在同一拍内 | 三连音 |
| 管乐长音 velocity < 70 | SusPiano |
| 管乐长音 70 <= velocity < 100 | SusMezzoForte |
| 管乐长音 velocity >= 100 | SusForte |

## 自动优化安全约束

- 先按起始时间把和弦归组；同拍和弦不自动写滑音、弱音或颤音，避免违反游戏的同拍奏法限制。
- 奏法只应用在高置信度的连接：滑音必须是 1-4 半音且前后没有重叠；颤音必须检测到返回原音的邻音往返。
- 每个乐句最多自动写入 3 次奏法，且两次自动奏法之间至少保留 2 个音符，防止整段音色被效果覆盖。
- 手工或导入后已有的有效 `ntype` 永远保留；不支持当前 BDO 乐器的 `ntype` 才会被清除并由转换检查标出。
- 钢琴延音踏板只用于真正跨到后续音符的长音，不会把所有长音都改成踏板。

## 不应自动处理的情况

- 不确定 BDO 实际音色的奏法：标签、多个颤音变体、基本、维持滤波器、滤波铜管。
- 电吉他 `FX(C2~G2)`，除非用户把轨道明确设为特效触发轨。
- 手鼓、钹、手碟完整音域未知时，不能标为 100% 转换通过。
- 单个轨道内旋律、和弦、打击效果混杂时，不应整轨应用同一种奏法，应按片段或音块推荐。

## 后续接入建议

1. FX 下拉框旁增加“推荐”按钮，只高亮候选，不修改音符。
2. 转换检查报告里增加“奏法建议”列，列出高置信度规则。
3. 时间轴中按音块显示建议标记，用户确认后再批量应用。
4. 每次用户在游戏内验证一种奏法，把结果补回本文件，再提升自动推荐等级。
